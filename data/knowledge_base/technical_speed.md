# Internet Speed Troubleshooting Guide

**Guide version:** 1.0

## Measurement requirements

Ask the customer to test with one device connected by Ethernet when possible, pause downloads and VPNs, and run two tests five minutes apart. Record download speed, upload speed, latency, test time, and whether the test used Wi-Fi or Ethernet.

Wi-Fi speed is affected by distance, walls, interference, and device capability; it is not sufficient by itself to confirm a line fault. Evening-only degradation should be recorded with times and repeated results.

## Troubleshooting

1. Check for a known outage or maintenance event.
2. Restart the router once using the Router Troubleshooting Guide.
3. Repeat a wired test with other devices paused.
4. If wired results remain below 70% of the plan's advertised speed in two tests, create a network-performance ticket.
5. If wired speed is normal but Wi-Fi is slow, follow the Wi-Fi Troubleshooting Guide.

## Playback error E-204

Error E-204 indicates that the app could not start the playback session. Confirm that the account is active and that the device is within the plan's device limit. Then close and reopen the app, sign out and back in, update the app to the latest available version, and retry on one title. Do not ask the customer to reinstall until these checks are complete.

If E-204 affects multiple titles after these steps, create a technical ticket containing the device type, app version, affected titles, and time of the latest attempt. If only one title is affected, create a content-availability ticket instead.

Do not promise a specific repair time unless an outage record or ticket provides it.
